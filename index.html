Safety AR Goggles

Our prototype goggles aim to promote safety in the school's Fabrication Laboratory (FabLab) and make FabLab users more independent and confident using the equipment.
The live feed from the ESP32 camera mounted on a bandsaw is read by the YoloV5 model and the model detects if the user's hand is getting too close to the saw, prompting a warning on the screen of the safety goggles along with haptic feedback.
To prepare the training data, we took over 300 images of users' hand on the bandsaw working on a project, and manually marked out where the hand is on the working area. I then fed the training data to YoloV5, allowing the model to output a warning to a server, alerting the user with visual and haptic warning. You can see more here.

Does this work on ARM

This website is a searcher for potential buyers of laptop with ARM architecture and Window OS - which does not support all softwares made for the typical x64 architecture - to check if the availability of the softwares they use daily, and if it is not available, the date that it would become available. This helps buyers to make informed decisions before purchasing an unfamiliar architecture system. This website was also meant to complement Windows on Arm Ready Software as we want to provide more than just games, such as Microsoft Suites, Adobe Creative Cloud Application, and more. Chatbot for Caregivers of people with autism

This customised ChatGPT provides caregivers of people with autism with support, tips, and resources. As this was made before CustomGPT was available widely for free, we collated relevant caregiver information from SGEnable in a knowledge retrieval system, and feed the relevant chunks to GPT-3.5, along with individualised prompt based on the user's input. Together with a team of 3 on the web development side of the project, we made the UI and UX to access the chatbot. The frontend was implemented using HTML, CSS, Bootstrap 5, while the backend connecting to the chatbot was made using Javascript.
